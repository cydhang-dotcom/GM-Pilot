# GM Pilot 设计规范 (Design Standards)

> **版本**: v2.2.0  
> **适用范围**: 移动端 Web App (React + Tailwind)

## 1. 核心布局体系 (Layout System)

### 1.1 轴心法则 (The 20px Axis)
*   **定义**: 在列表、时间轴、详情页中，核心视觉引导线（图标中心线或连接线）距左边缘 **20px** (5rem)。
*   **应用**:
    *   **Timeline**: 连接线中心坐标 = 20px。
    *   **Social Dynamic**: 增减员卡片头像左边缘对齐 20px 轴线。

### 1.2 穿透层级 (Drill-down Hierarchy)
*   **L1**: 底部导航主页。
*   **L2**: 业务看板/概览列表。
*   **L3**: 业务详情（如薪资总表、员工全量档案）。
*   **L4**: 原子明细（如个人工资条、单一扣税凭证）。采用 `DetailLayout` 的 Overlay 动画进行层级推入。

## 2. 交互组件规范

### 2.1 状态与日期 (Status & Dates)
*   **办理中 (Processing)**: 
    *   颜色: `orange-600`。
    *   动效: `animate-pulse` 呼吸效果。
    *   标签: 必须包含“预计”字样及预计日期（如：预计12-18）。
*   **已完成 (Done)**: 
    *   颜色: `emerald-600`。
    *   标签: 必须显示确切的办理日期（如：2023-12-05）。

### 2.2 拟物化视觉
*   **凭证/发票**: 使用虚线分割线及左右圆角切口模拟纸质撕票感。
*   **文件预览**: PDF 预览卡片需具备 24px 以上的大尺寸阴影（Shadow-2xl）以体现层叠感。

## 3. 隐私模式 (Privacy Standard)
*   **全局开关**: 位于 Company 页。
*   **表现**: 敏感数值（如个人工资、银行余额）默认应用 `blur-md`。
*   **局部切换**: 眼睛图标必须位于数值右侧，点击后 300ms 解除模糊。
