# 功能需求: 交付进度时间轴 (Timeline)

> **所属模块**: Inbox (核心交付件)
> **优先级**: P0
> **设计核心**: 轴心锁定 (Pixel Axis)、拟物状态 (Skeuomorphic Status)

## 1. 用户故事 (User Story) 与 数据逻辑

### US1: 履约上帝视角
*   **数据业务逻辑**:
    *   **状态映射**: `Completed` (有结束日), `Active` (进行中), `Pending` (未开始)。
    *   **自动收起**: 历史已完成节点超过 5 个时默认折叠。

## 2. 界面行为规范 (UI Behaviors)

*   **20px 轴心锁定标准**: 
    *   连接线的 X 坐标中心点固定在 **20px** 处。
    *   Marker 的左边缘起始于 10px 处（中心对齐轴线）。
    *   卡片标题起始于轴线右侧 32px 处（即 52px 坐标）。
*   **节点动效**: `Active` 节点需具备 2000ms 周期的呼吸扩散阴影。

## 3. 验收标准 (Acceptance Criteria)

- [x] 轴线在不同分辨率设备上均需保持距离左边缘 20px。
- [x] 点击带“待确认”标签的节点，必须能直接跳转至对应的 Overlay。
