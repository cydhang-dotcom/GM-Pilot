
# 业务需求: 员工管理 - 离职手续 (Employee Offboarding)

> **关联 ID**: `hr-emp-off`
> **优先级**: P1
> **设计核心**: 流程合规 (Compliance)、资产交接 (Handover)、风险提示 (Risk Awareness)

## 1. 用户故事 (User Stories) 与 数据逻辑

### US1: 规范化减员流程
*   **故事**: 作为总经理，当员工离职时，我需要在手机上一键触发减员申报，防止产生多余的社保费用。
*   **数据业务逻辑**:
    *   **申报截止日校验**: 若离职日期超过本月社保申报截止日，提示 GM 该员工下月仍会产生一笔基础社保费。
    *   **权限回收**: 接口触发后，需同步通知 IT 系统注销该员工的邮箱及 OA 账号。

## 2. 界面行为规范 (UI Behaviors)

*   **警示色彩**: 
    *   使用 `Rose-600` (深红色) 作为主操作色，暗示操作的严肃性与不可逆性。
*   **Checklist 强制性**: 
    *   “工作交接确认” 勾选项为必选，否则“确认离职”按钮处于 `Disabled` 状态。

## 3. 验收标准 (Acceptance Criteria)

- [x] 必须包含最后工作日选择器。
- [x] 必须包含离职原因分类下拉框。
- [x] 必须展示该操作导致的后续系统行为（社保停缴、考勤结算等）。
- [x] 完成后，该员工在 `EmployeeList` 中的状态需变更为“已离职”并移入历史库。
