# 业务需求: 员工管理 - 档案详情 (Employee Profile)

> **入口 ID**: `hr-emp`
> **优先级**: P1
> **设计核心**: 档案全景 (Panoramic View)、看板筛选 (Stats Filter)、原位脱敏 (In-place Privacy)

## 1. 业务功能概览

### 1.1 看板筛选逻辑 (Dashboard Stats Filter)
*   **交互故事**: GM 进入员工管理页后，希望通过顶部的统计卡片快速过滤出对应的员工群体，而不需要去搜索框慢慢输入。
*   **状态引擎**:
    *   **在职员工 (Total)**: 默认视图，排除 `Status == '离职'` 的所有人员。
    *   **本月入职 (New Hires)**: 筛选 `joinDate` 属于当前自然月的人员。
    *   **本月离职 (Resigned)**: 筛选 `Status == '离职'` 且离职日期属于当前月的人员。
    *   **试用期 (Probation)**: 筛选 `Status == '试用'` 的人员。
*   **反馈标识**:
    *   选中某个统计块后，该块应呈现反色高亮（品牌色实色背景）。
    *   列表上方展示“正在查看: [筛选标签]”提示，并提供“清空筛选”快捷按钮。

### 1.2 档案全景图谱 (Data Sections)
*   **核心身份 (Identity Header)**: 顶部渐变背景，展示“在职/离职”标签及“已实名”图标。
*   **任职信息 (Appointment)**: 部门、岗位、入职日期、试用期状态。
*   **劳动合同 (Contract)**: 点击穿透至合同管理，展示司龄勋章。
*   **个人全量资料 (Personal Details)**: 隐私字段默认模糊（如证件号）。

## 2. 界面行为规范 (UI Behaviors)

*   **20px 轴心锁定**: 
    *   统计卡片内部布局逻辑对齐。
    *   员工列表头像左边缘对齐 **20px** 轴线。
*   **筛选动效**: 
    *   点击统计块后，列表内容执行 `Fade-In` 动画刷新。
    *   统计块高亮切换具备 `300ms` 的平滑过渡。
*   **模态编辑**: 点击编辑图标后，通过 `SectionModal` 从底部弹出，支持原位修改保存。

## 3. 验收标准 (Acceptance Criteria)

- [x] 点击顶部“本月入职”统计块，下方列表必须仅显示本月入职员工。
- [x] 点击“本月离职”统计块，必须能看到已标记为离职状态的员工及其离职日期。
- [x] 筛选状态下必须有明显的视觉提示，告知用户当前正在查看的子集。
- [x] 司龄勋章必须根据入职日期动态计算显示。
- [x] 隐私模式必须生效，证件号码初次加载为模糊状态。

## 4. API 接口

| 接口描述 | Endpoint |
| :--- | :--- |
| 获取员工概览统计 | `/api/hr/employees/stats` |
| 获取过滤后的员工列表 | `/api/hr/employees?filter={type}&q={search}` |
