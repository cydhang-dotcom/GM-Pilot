# 业务需求: 员工管理 - 档案详情 (Employee Profile)

> **入口 ID**: `hr-emp`
> **优先级**: P1
> **设计核心**: 信息分级 (Layering)、原位脱敏 (In-place Privacy)、业务穿透 (Cross-Navigation)

## 1. 字段展示策略 (Display Strategy)

### 1.1 精简预览 (Card Previews)
为保持页面高效，预览态仅保留核心业务字段：
- **[1] 任职信息**: 部门、岗位、入职日期。
- **[2] 劳动合同**: 展示最新的一份合同摘要 [类型、期限、到期日]。
    - **穿透逻辑**: 包含一个“查看详细合同履历”入口，点击后跳转至 `hr-6` (合同管理) 模块。
- **[3] 基本信息**: 性别、生日、证件号。
- **[4] 户籍居住**: 户口性质、现居住地。
- **[5] 教育/ [6] 经历**: 20px 轴心时间轴预览。
- **[7] 联系方式**: 手机、邮箱。
- **[8] 紧急联系人**: 姓名、关系、电话。
- **[9] 银行发放**: 脱敏银行卡、开户行。

### 1.2 穿透导航规范 (Navigation)
- 档案详情页中的“劳动合同”模块不再仅仅是数据展示，而是作为进入“合同管理”深层业务的快捷入口。
- 点击跳转行为应具有平滑的页面切换感（Slide-out/In）。

## 2. 界面行为规范 (UI Behaviors)

*   **20px 轴心**: 经历类列表的时间轴圆点中心点锁定在容器左边缘 **20px** 处。
*   **原位脱敏**: 证件/银行卡号数值右侧配备“眼睛”图标切换可见性。

## 3. 验收标准 (Acceptance Criteria)
- [x] 合同模块必须包含“全景记录”跳转入口。
- [x] 点击跳转入口必须能正确路由至 `/work/hr-6`。
- [x] 预览态数据必须显示最新的合同期限及到期日。
