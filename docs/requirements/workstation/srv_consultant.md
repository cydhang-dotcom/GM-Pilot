# 业务需求: 专属顾问与服务台 (Consultant & Tickets)

> **入口 ID**: `ot-1 ~ ot-5`
> **优先级**: P2
> **设计核心**: 极简触达 (Quick Reach)、闭环反馈 (Loop)

## 1. 用户故事 与 数据逻辑

### US1: 顾问直连 (Concierge)
- **视觉**: 顶部展示顾问头像、姓名、角色（CSM/CPA）及联系电话。
- **操作**: 快捷按钮支持一键调起拨号或跳转邮件。

### US2: 服务矩阵 (Service Matrix)
- **分类**: 政策咨询、员工服务、口径确认、异常处理。
- **状态**: 采用 2x2 Grid 展示，点击切换下方历史工单列表。

### US3: 工单聊天流 (Ticket Chat)
- **逻辑**: GM 发起请求 -> 顾问回复 -> GM 确认或追问。
- **评价**: 工单办结后展示“星级评价”入口。

## 2. 行为规范

*   **对话气泡**: GM 气泡为 `Indigo-600` (右侧)，顾问气泡为 `White` (左侧)。
*   **轴心锁定**: 服务记录列表标签对齐 **20px 轴线**。

## 3. 验收标准
- [x] 顶部顾问卡片必须显示实时可用的联系方式。
- [x] 状态为“处理中”的工单必须有呼吸动效标识。
- [x] 必须支持带上下文的对话历史查看。
