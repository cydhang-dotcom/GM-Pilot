# 功能需求: 对账中心 (Reconciliation)

> **入口 ID**: `fn-rec`
> **优先级**: P0
> **定位**: 财务模块的核心入口，连接流水、发票、报销与凭证。
> **涉及页面**: `pages/workstation/finance/Reconciliation.tsx`

## 1. 核心看板 (Health Dashboard)
- [x] **健康度评分**: 
  - 大字展示当前同步健康度百分比 (如 92%)。
  - 状态标签: 标记“正常”或“异常”。
  - 显示最后同步时间。
- [x] **关键指标**:
  - **本月流水**: 展示已同步的银行流水笔数。
  - **待处理差异**: 红色高亮显示需要总经理介入的差异数量。
- [x] **手动同步**: 提供刷新按钮，手动触发银行/税局数据同步。

## 2. 差异处理清单 (Action List)
- [x] **聚合展示**: 
  - 将同类异常（如“流水未解释”、“发票待归属”）聚合为单行展示。
  - **计数角标**: 红点标记该类别的待处理数量。
- [x] **优先级标记**: 
  - **Urgent (红)**: 阻碍关账的紧急事项（如发票缺失）。
  - **Warning (橙)**: 一般性确认事项（如补充说明）。
- [x] **列表操作**: 每行提供快捷“处理”按钮。

## 3. 差异详情与沟通 (Level 3 Overlay)
- [x] **异常卡片**: 
  - 顶部展示醒目的警示图标与异常类型。
  - 文字说明异常的具体原因与处理建议。
- [x] **上下文数据 (Context)**: 
  - 自动关联展示相关的原始数据快照（如银行流水的金额、时间、对手方），辅助回忆。
- [x] **沟通记录 (Chat)**:
  - **代理留言**: 展示代理会计对该笔异常的询问或备注。
  - **快速回复**: 底部提供输入框，允许总经理直接回复说明情况。
- [x] **一键处理**: 
  - 底部固定操作栏，提供“标记已处理”或具体的业务动作按钮。
