
# 业务需求: 员工管理 - 入职录入 (Employee Onboarding)

> **关联 ID**: `hr-emp-add`
> **优先级**: P1
> **设计核心**: 强制合规 (Compliance)、联动智能 (Intelligent Linkage)、极简录入 (Streamlined)

## 1. 用户故事 (User Stories) 与 数据逻辑

### US1: 动态增员与合同发起 (Onboarding & Contract)
*   **故事**: 我需要在手机上快速录入新入职人员信息，同时确定其薪资标准、社保基数，并一键发起电子合同签订。
*   **数据业务需求**:
    *   **全流程录入 (5步法)**:
        1.  **员工信息**: 姓名/身份证/手机 + **部门/岗位/入职日期**。
            *   **逻辑联动**: 部门、岗位、入职日期为**永久必填项**，确保入职时岗职信息明确。
        2.  **薪酬核定**: 
            *   **核定月薪 (税前)**: **必须填写 (Required)**。
        3.  **合同信息**: 
            *   **发起合同签订**: 默认关闭。
        4.  **个税申报**: 
            *   默认勾选“申报个税”。
            *   **智能联动**: “个税核定基数”默认等于“核定月薪”。点击后弹出 **Modal** 进行手动修改。
        5.  **社保公积金**: 
            *   默认勾选“办理参保”。
            *   **智能联动**: “社保基数”与“公积金基数”默认等于“核定月薪”。点击后弹出 **Modal** 编辑。

### US2: 员工自助录入 (Self-Onboarding)
*   **故事**: 录入工作量大时，我希望生成一个二维码，让员工自己扫码填好基础信息（证件、银行卡等），我只负责最后审核确认。
*   **数据业务逻辑**:
    *   **临时凭证**: 生成的二维码包含带时效的 Token。
    *   **数据流转**: 员工填写后，数据进入“待审批入职”队列。

## 2. 界面行为规范 (UI Behaviors)

*   **录入表单**:
    *   采用 **分段式卡片 (Sectioned Cards)** 布局，明确区分 1-5 步骤。
    *   底部固定按钮文案为“**确认入职**”。
    *   必填项在标签后显示红色 `*`。
*   **二维码入口**:
    *   位于“1. 员工信息”区域的右上角。
    *   点击后弹出全屏模糊背景的 Modal，展示大尺寸二维码及**企业基本信息（名称/Logo）**。
*   **基数编辑**: 采用中心位浮层 Modal，背景应用 `backdrop-blur`。

## 3. 验收标准 (Acceptance Criteria)

- [x] “核定月薪”字段为强制输入项，不得为空。
- [x] 岗位/部门/入职日期为强制输入项，作为档案核心。
- [x] 个税、社保、公积金基数初始值必须跟随月薪自动变动。
- [x] 必须提供“员工自助扫码”入口。
- [x] 二维码 Modal 必须包含清晰的操作提示文案。
- [x] 二维码 Modal 必须展示企业名称与 Logo，增强信任感。
- [x] Modal 内必须包含“恢复同步月薪”按钮。

## 4. API 接口 (API Interfaces)

### 4.1 新增员工 (Add Employee)
*   **Endpoint**: `POST /api/hr/employees`
*   **输入参数 (Request)**: 同 `hr_employee.md` 定义，此处专注于录入校验。
