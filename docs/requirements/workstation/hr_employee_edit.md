
# 业务需求: 员工管理 - 资料修改 (Employee Edit)

> **关联 ID**: `hr-emp-edit`
> **优先级**: P1
> **设计核心**: 信息连贯 (Coherent)、快速响应 (Quick Edit)

## 1. 用户故事 (User Stories) 与 数据逻辑

### US1: 关键信息动态调整
*   **故事**: 员工晋升或调岗后，我需要快速在手机上修改其部门、岗位和薪资标准。
*   **数据业务逻辑**:
    *   **历史留痕**: 薪资和岗位的变更必须在后台记录 `Version_History`，但前端仅展示最新版本。
    *   **联动处理**: 修改月薪后，若个税或社保基数原本为“同步月薪”状态，则自动触发重算。

## 2. 界面行为规范 (UI Behaviors)

*   **表单交互**: 
    *   复用 `EmployeeAdd` 的 Input 样式，但页面主色调采用更稳重的蓝色调（而非新增时的 Indigo）。
*   **反馈提示**: 
    *   底部保存按钮需具备 500ms 的 `Loading` 状态，模拟后端同步。

## 3. 验收标准 (Acceptance Criteria)

- [x] 进入编辑页时，必须预填充员工当前的所有数据。
- [x] 月薪修改提示框必须明确告知其对财务报表的影响。
- [x] 保存成功后必须返回详情页并刷新数据。
