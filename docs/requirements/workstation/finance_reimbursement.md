# 功能需求: 费用报销 (Reimbursement)

> **入口 ID**: `fn-reim`
> **优先级**: P1
> **涉及页面**: `pages/workstation/finance/Reimbursement.tsx`

## 1. 发起报销 (Initiate)
- [x] **智能入口卡片**: 
  - 顶部展示渐变色大卡片“发起报销”。
  - 强调“拍票据 · 自动识别 · 智能归集”的便捷性。
  - 右侧悬浮“+”号按钮，点击触发拍照/上传。

## 2. 报销列表 (List)
- [x] **单据概览**: 展示报销摘要（如“11月北京出差”）、报销人、时间及总金额。
- [x] **合规预检状态**:
  - **Evidence Missing (红)**: 证据链缺失（如缺支付凭证），需用户补齐。
  - **Approved (绿)**: 审批通过/合规。
- [x] **缺失提示**: 在列表卡片内直接用红字提示具体的缺失项（如“补: 支付凭证”）。

## 3. 报销详情 (Level 3 Overlay)
- [x] **头部金额**: 醒目展示报销总额及报销人头像。
- [x] **异常警示 (Alert)**: 
  - 若存在合规问题，顶部显示红色警示卡片。
  - 明确列出系统检测到的缺失材料（如“缺少支付截图”）。
- [x] **费用明细**:
  - 列表展示具体的消费项（交通、住宿、餐饮）。
  - 显示每项的金额及对应的图标。
- [x] **附件证据**: 
  - 缩略图列表展示已上传的票据、行程单、截图。
- [x] **补全操作**: 
  - 针对异常单据，底部固定“补充缺失材料”按钮（红色强调）。
