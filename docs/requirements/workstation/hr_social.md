# 业务需求: 五险一金 (Social Security & Fund)

> **入口 ID**: `hr-4`  
> **优先级**: P0 (核心成本项)  
> **设计核心**: 办理透明 (Process Transparency)、变动敏感 (Change Sensitivity)、合规校验 (Compliance)

## 1. 用户故事 与 数据逻辑

### US1: 办理进度看板 (Execution Oversight)
*   **故事**: 作为总经理，我需要一眼看到本月新增的人员是否已经办完社保，没办完的预计什么时候能办好。
*   **数据业务逻辑**:
    *   **状态引擎**:
        *   `Status == '办理中'`: 显示“预计办理: MM-DD”，文本颜色为 `orange-600`，带呼吸动效。
        *   `Status == '已完成'`: 显示“办理日期: YYYY-MM-DD”，文本颜色为 `emerald-600`。
    *   **动态看板**: Dashboard 的“增减员动态”聚合卡片下方需显示最近一条变动的具体进度描述。

## 2. 界面行为规范 (UI Behaviors)

*   **视觉标准**: 
    *   增减员动态列表卡片左侧需有颜色标识条（新增为 Blue，减少为 Slate）。
    *   办理中状态的 `Clock` 图标需有 `animate-pulse` 效果。
*   **轴心锁定**: 列表项头像及标识图标严格对齐 **20px 轴线**。

## 3. 验收标准

- [x] 增减员详情列表必须展示每个人的确切办理日期或预计日期。
- [x] 办理中状态的人员必须有橙色高亮并伴随呼吸动画。
- [x] 二级详情页必须支持穿透查看单人的社保比例拆解。
- [x] 必须提供 PDF 拟物化的“回单预览”功能。
