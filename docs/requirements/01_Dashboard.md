# 模块需求: 经营总览 (Dashboard)

> **优先级**: P0
> **涉及页面**: `pages/Dashboard.tsx`
> **设计风格**: Financial Pilot (盈亏视角, 决策导向)

## 1. 概述
Dashboard 是总经理的“经营驾驶舱”，从单纯的支出记录转变为 **盈亏视角 (Profit & Loss)**。核心目标是让 GM 一眼看懂本月赚了多少、钱花哪了、有无风险。

## 2. 功能组件拆解

### 2.1 顶部导航与周期切换 (Header & Toggle)
- [x] **标题更名**: 由“企业驾驶舱”更名为“经营总览”，视觉更通透。
- [x] **周期切换**: 提供月份切换器 (如 `2023-12` vs `2024-01`)，支持不同经营周期的快速对比。

### 2.2 核心盈亏看板 (Hero Card)
- [x] **盈亏视觉**: 
  - **盈利**: 翡翠绿。
  - **亏损**: 玫瑰红。
- [x] **净利率监控**: 显示 Net Profit Margin (%)。

### 2.3 资金流走势 (Cash Flow)
- [x] **隐私默认**: 余额默认打码（Blur 效果），点击眼睛切换。
- [x] **环比对比**: 本月（实线）vs 上月（虚线）走势图。

### 2.4 智能风险诊断 (Smart Risk Monitor) - *AI Chat Style*
- [x] **交互重构**: 
  - **无按钮设计**: 移除原有的展开/收起按钮。
  - **固定高度**: 折叠状态固定 4 行高度（约 88px），防止打字时页面抖动。
  - **省略号入口**: 文末显示紫色闪烁省略号 (`...`)，点击文本区域或省略号即可平滑展开全文。
- [x] **打字机特效**: 诊断内容逐句渲染，增强 AI 在场感。

### 2.5 经营支出结构 (Expenditure Structure)
- [x] **全扁平化**: 弃用 Pie Chart，统一采用 **线性进度条列表**。
- [x] **结构对齐**: 分类标签、百分比进度条、具体金额需在视觉上与收入构成模块高度对称。

### 2.6 收入回款明细 (Revenue Breakdown)
- [x] **状态识别**: 左侧图标区分“已入账”（绿勾）与“待入账”（闹钟）。
