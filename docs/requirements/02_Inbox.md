# 模块需求: 智能待办 (Inbox)

> **优先级**: P0
> **涉及页面**: `pages/Inbox.tsx`

## 1. 概述
Inbox 是管理者的“决策中心”，聚合了所有需要确认、签字、支付的任务，以及关键交付节点的进度监控。

## 2. 功能组件拆解

### 2.1 交付进度时间轴 (Timeline) [Moved from Dashboard]
按时间顺序展示本月关键业务节点（如发薪、报税），让用户清晰了解当前进度。
> [📄 查看详细需求: 交付进度时间轴](./dashboard/timeline.md)
- [x] **智能折叠**: 默认仅展示进行中节点。
- [x] **关键指标**: 在时间轴卡片内直接展示薪资、人员变动等核心数据。
- [x] **状态交互**: 点击节点可跳转至对应模块。

### 2.2 账单置顶卡片 (Pinned Bill)
针对服务费、大额税金等紧急支付任务的特殊展示形态。
- [x] **详情交互**: 点击卡片弹出 `BillDetail` (Level 3 Overlay)，展示账单明细并提供“立即支付”按钮。

### 2.3 任务聚合与列表 (Task Grouping)
解决任务碎片化问题，将同类任务（如入职确认）聚合展示。
- [x] **OA 聚合**: 点击“OA 审批”卡片进入二级聚合页。
- [x] **入职聚合**: 点击“入职任务”卡片进入二级聚合页，统一管理多名候选人的入职流程。

### 2.4 工资发放确认 (Salary Confirmation)
针对月度薪资发放的高优任务。
- [x] **详情交互**: 点击工资发放任务弹出 `SalaryDetail` (Level 3 Overlay)。
- [x] **资金透视**:
  - 醒目展示实发总额与涉及人数。
  - 展示账户余额是否充足。
  - 列出资金构成（基本工资、五险一金、个税）。
- [x] **风险提示**: 若存在试用期员工或其他异常，显示橙色警示卡片。
- [x] **决策操作**: 底部提供“退回修改”与“确认并发放”按钮。

### 2.5 OA 审批详情 (OA Workflow)
内部 OA 审批流的专项处理视图。
- [x] **详情交互**: 点击单条审批任务弹出 `OADetail` (Level 3 Overlay)。
- [x] **审批操作**: 在详情页底部提供“同意”与“驳回”操作栏。

### 2.6 入职确认详情 (Entry Workflow)
- [x] **详情交互**: 点击单条入职任务弹出 `EntryDetail` (Level 3 Overlay)。
- [x] **材料核验**: 展示 Offer、身份证、学历证书等材料的核验状态。
- [x] **归档操作**: 底部提供“确认并归档”按钮。
