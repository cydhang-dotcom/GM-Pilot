# 模块需求: 智能待办 (Inbox)

> **优先级**: P0
> **涉及页面**: `pages/Inbox.tsx`

## 1. 概述
Inbox 是管理者的“决策中心”，聚合了所有需要确认、签字、支付的任务。

## 2. 功能组件拆解

### 2.1 账单置顶卡片 (Pinned Bill)
针对服务费、大额税金等紧急支付任务的特殊展示形态。
- [x] **详情交互**: 点击卡片弹出 `BillDetail` (Level 3 Overlay)，展示账单明细并提供“立即支付”按钮。

### 2.2 任务聚合与列表 (Task Grouping)
解决任务碎片化问题，将同类任务（如入职确认）聚合展示。
- [x] **OA 聚合**: 点击“OA 审批”卡片进入二级聚合页（Level 2 Group View），在当前页面内切换视图，不跳转路由。

### 2.3 OA 审批详情 (OA Workflow)
内部 OA 审批流的专项处理视图。
- [x] **详情交互**: 点击单条审批任务弹出 `OADetail` (Level 3 Overlay)。
- [x] **审批操作**: 在详情页底部提供“同意”与“驳回”操作栏。
