# 业务需求: 常用业务菜单 (Menu Groups)

> **所属模块**: Company
> **优先级**: P1
> **设计核心**: 信息层级 (Hierarchy)、资料中心 (Doc Hub)

## 1. 用户故事 (User Stories) 与 数据逻辑

### US1: 资料管家一站式 (One-stop Docs)
*   **故事**: 无论是开票地址还是公章状态，我需要在一个页面通过简单的分类找到它们。
*   **数据业务逻辑**:
    *   **业务映射**: 强制将功能分为“常用业务资料”、“权益与资产”、“企业管理”三个逻辑组。
    *   **实时状态反馈**: 资产类项（如“电子印章”）需在菜单右侧实时显示 `Value` (如：3枚)。

## 2. 界面行为规范 (UI Behaviors)

*   **20px 轴心锁定**: 
    *   所有菜单项的图标中心点必须锁定在 **20px** 轴线。
    *   菜单分组标题的起始位置需对齐图标左边缘。
*   **Overlay 穿透**: 
    *   点击任意菜单项，均进入 Level 3 全屏详情 Overlay。

## 3. 验收标准 (Acceptance Criteria)

- [x] 菜单图标颜色必须与 `DesignStandards.md` 中的语义色保持一致。
- [x] 点击“开票信息”后的详情页必须支持一键复制所有核心字段。
- [x] 资产有效期预警必须以橙色文字显示在菜单副标题位置。
