# 业务需求: 企业数字名片 (Identity)

> **所属模块**: Company
> **优先级**: P1
> **设计核心**: 品牌识别 (Branding)、快速传递 (Swift Sharing)

## 1. 用户故事 (User Stories) 与 数据逻辑

### US1: 统一代码快速调用 (Quick Code Copy)
*   **故事**: 面对合作伙伴，我需要能一秒钟找出公司的税号并完成复制。
*   **数据业务逻辑**:
    *   **脱敏逻辑**: 默认显示前 6 位 + 尾 4 位，点击“小眼睛”或“复制”时获取全量 `Credit_Code`。
    *   **状态维护**: 实时根据 `Enterprise_Registry` 拉取“实名认证”与“经营状态”。

## 2. 界面行为规范 (UI Behaviors)

*   **拟态卡片设计**: 
    *   名片高度锁定在 `200px`，背景采用品牌色渐变。
*   **20px 轴心锁定**: 
    *   左侧 Logo 的圆角矩形容器中心点锁定在 **20px** 轴线。
*   **动效**: 
    *   点击“复制”成功后，按钮需呈现 1000ms 的 `Success_Check` 反馈。

## 3. 验收标准 (Acceptance Criteria)

- [x] 税号卡片必须具备一键复制图标。
- [x] 认证标签（Verified）必须与企业全称在同一水平线上展示。
- [x] 二维码预览必须采用全屏居中 Modal 展示。
