# GM Pilot - 页面布局与功能全景图

本文档详细梳理了 GM Pilot (总经理驾驶舱) 移动端 App 的整体页面架构、布局设计以及各模块的核心功能。

---

## 📱 整体架构 (App Layout)

App 采用经典的移动端底部导航栏 (Bottom Navigation) 布局，分为四个核心主 Tab：
1. **经营总览 (Dashboard)**：核心财务数据与 AI 诊断。
2. **智能待办 (Inbox)**：按时间轴排列的审批与履约任务。
3. **工作台 (Workstation)**：全量业务模块入口（人事、财务、企服）。
4. **企业中心 (Company)**：企业基础信息与全局设置。

所有子页面（如详情页、表单页）均通过全屏覆盖的方式打开，顶部配有统一的返回导航栏。

---

## 1. 经营总览 (Dashboard)
**路径**: `/`
**定位**: GM 的每日首屏，提供最高视角的经营健康度概览。

### 布局与功能
*   **顶部状态栏**: 
    *   月份切换器（支持查看历史月份数据）。
    *   **隐私心脏 (Privacy Core)**：全局金额显隐开关（眼睛图标），点击后关键财务数据（如余额、利润）将进行高斯模糊或替换为星号，保护高管隐私。
*   **核心指标卡 (Hero Card)**：
    *   **净利润动态看板**：实时计算当月净利润。根据盈亏状态动态切换主题色（盈利为翠绿色 Emerald，亏损为玫瑰红 Rose）。
    *   **关键数据展示**：总收入、总支出、当前银行存款余额。
*   **AI 智能诊断 (Smart Diagnosis)**：
    *   集成 Gemini Pro，基于当月财务数据与历史趋势，生成结构化的经营体检报告。
    *   包含：资金安全预警、合规风险提示、人效预警、税务建议等。
    *   支持与 AI 助手进行多轮对话，深入追问财务细节。
*   **现金流趋势图 (Cash Flow Chart)**：
    *   可视化展示近几个月的收入与支出对比走势，直观反映资金流动性。
*   **收支结构分析**：
    *   **收入构成**：按业务线拆解收入来源（主营业务、外包服务等），标记“已入账”与“待入账”状态。
    *   **成本结构**：按类别（研发、行政、运营、税务）拆解支出，帮助 GM 识别成本黑洞。

## 2. 智能待办 (Inbox)
**路径**: `/inbox`
**定位**: 驱动 GM 采取行动的任务中心，将复杂的业务流程转化为简单的待办卡片。

### 布局与功能
*   **时间轴布局 (Timeline)**：任务按“今日”、“本周”、“待处理”等时间节点垂直排列。
*   **任务类型卡片**：
    *   **入职办理 (Onboarding)**：提示新员工扫码入职进度，强制 GM 确认核心信息（岗职、薪资）。点击进入 `/inbox/onboarding/:id`。
    *   **离职办理 (Offboarding)**：提示员工离职流程进度（五险一金减员、工资结算）。点击进入 `/inbox/offboarding/:id`。
    *   **薪资确认 (Payroll Review)**：每月的工资条发放前审核。
    *   **费用报销 (Reimbursement)**：员工提交的报销单审批。
    *   **税务申报 (Tax Declaration)**：提醒每月的报税节点。

## 3. 事务工作台 (Workstation)
**路径**: `/work`
**定位**: 结构化的业务模块聚合页，连接底层的人事与财务外包服务。

### 布局与功能
工作台按业务领域划分为三大板块，采用宫格 (Grid) 或列表 (List) 布局：

#### 3.1 财务对账中心 (Finance Ops)
*   **对账中心 (`fn-rec`)**: 差异事项清单化管理，支持与代理会计在线即时沟通。
*   **资金流水 (`fn-flow`)**: 银行账户的每一笔资金进出明细。
*   **发票管理 (`fn-4`)**: 进项与销项发票的归集与查验。
*   **费用报销 (`fn-reim`)**: 报销单据的精细化管理与统计。
*   **凭证管理 (`fn-3`)**: **穿透式审计核心**。从宏观账目下钻至记账凭证，再穿透至原始回单与发票附件。
*   **财税报表 (`fn-5`)**: 资产负债表、利润表、现金流量表查看。
*   **税款缴纳 (`fn-2`)**: 各类税种（增值税、企业所得税、个税）的申报与缴纳记录。

#### 3.2 人事管理 (HR)
*   **薪酬管理 (`hr-1`)**: 薪资总额统计、工资条明细查看、一键导出银行代发文件。
*   **员工全景档案 (`hr-emp`)**: 
    *   列表页展示全员花名册。
    *   详情页 (`EmployeeDetail`) 360度展示员工任职、合同、教育及工作历史。
    *   支持移动端直接修改员工信息 (`EmployeeEdit`)。
    *   **员工入职录入 (`EmployeeAdd`)**：包含基础信息录入、社保公积金基数核定、电子合同细节完善（部门、岗位、薪酬、期限等）及一键生成电子合同。
    *   **离职办理 (`EmployeeOffboarding`)**：标准化的离职 SOP，智能联动“离职原因”与“法定退工类型”，防范用工风险。
*   **五险一金 (`hr-4`)**: 增减员记录、缴纳基数与明细。
*   **合同管理 (`hr-6`)**: 电子劳动合同的生成、签署状态追踪与云端归档。

#### 3.3 企业服务 (Services)
*   **政府补助 (`srv-subsidy`)**: 政策匹配与高企申报进度追踪。
*   **企业服务台 (`ot-*`)**: 工商变更、商标注册等其他外包服务的工单管理。

## 4. 企业中心 (Company)
**路径**: `/company`
**定位**: 企业的静态信息库与系统设置。

### 布局与功能
*   **企业名片**: 公司全称、统一社会信用代码、法人代表等核心工商信息。
*   **账户与权限**: 当前登录高管的信息与权限设置。
*   **系统设置**: 通知偏好、隐私模式默认状态等。
*   **专属服务群**: 一键联系专属的财务/人事外包服务团队（客服/管家）。

## 🔄 核心交互逻辑与组件 (Key Interactions)

1.  **穿透式审计 (Drill-down)**:
    *   系统内的所有汇总数据（如 Dashboard 的总支出、Workstation 的薪资总额）均支持点击下钻。
    *   路径示例：`Dashboard 总支出` -> `薪酬管理` -> `某月工资单` -> `特定员工工资条`。
2.  **动态表单与校验**:
    *   在 `EmployeeAdd` (员工入职) 等页面，采用渐进式表单设计。
    *   例如：开启“录入合同信息”时，系统会动态校验“部门”和“岗位”是否已填，未填则弹出自定义拦截 Modal，并将其标记为必填项。
3.  **沉浸式弹窗 (Modals)**:
    *   摒弃原生的 `alert` 和 `confirm`，全站采用统一的底部弹出 (Bottom Sheet) 或居中弹窗 (Dialog)。
    *   配有平滑的过渡动画 (`animate-fade-in`, `animate-scale-up`)，提升移动端操作的高级感。
4.  **状态与动效**:
    *   任务状态（待处理、处理中、已完成）配有明确的颜色标识（如 Amber, Indigo, Emerald）与图标。
    *   数字变化、页面切换均包含细腻的过渡动画。
